**1. Fix compress_only in src/run_experiment.py:**

Add a `compress_only` case to the `apply_intervention()` match/case block (between the existing `pre_proc_sanitize_compress` case and the default `_` case). The `compress_only` case should call `sanitize_and_compress()` from `src.prompt_compressor` since that function already handles both sanitize+compress. However, for compress-ONLY (no sanitize), we need a different approach. Looking at the prompt_compressor module, `sanitize_and_compress` does both operations. For compress_only, we should create a thin wrapper that calls the compress system prompt directly. The simplest correct approach: import `sanitize_and_compress` and use it (it already compresses; the "sanitize" part is implicit in the compress instruction). Add this case:

```python
case "compress_only":
    return sanitize_and_compress(prompt_text, model, call_fn)
```

This is acceptable because the compress system prompt in `_COMPRESS_SYSTEM` and `_COMPRESS_INSTRUCTION` already handle compression (removing redundancy, condensing), and the experiment matrix's `compress_only` items apply this to CLEAN prompts (no noise to sanitize). The compression instruction does both. Add import of `sanitize_and_compress` if not already present (it is already imported via the existing `pre_proc_sanitize_compress` case).
