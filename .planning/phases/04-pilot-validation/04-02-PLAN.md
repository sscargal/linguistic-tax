**compute_cost_projection(conn, pilot_prompt_ids, n_full_prompts=200, n_bootstrap=10000, confidence_level=0.95, output_path="results/pilot_cost_projection.json") -> dict:**

1. Query all completed pilot runs
2. Aggregate cost per prompt: for each pilot prompt_id, sum `total_cost_usd` across all its runs
3. Build numpy array of per-prompt costs (length = number of pilot prompts)
4. Compute scale_factor = n_full_prompts / len(pilot_prompt_ids)
5. Use `scipy.stats.bootstrap((per_prompt_costs,), np.sum, n_resamples=n_bootstrap, confidence_level=confidence_level, method="bca")` to get CI
6. Scale CI bounds by scale_factor
7. Also compute per-condition breakdown: group by (model, intervention, noise_type), sum costs, scale each
8. Build output dict:
   ```python
   {
       "generated_at": ...,
       "pilot_prompt_count": len(pilot_prompt_ids),
       "full_prompt_count": n_full_prompts,
       "pilot_total_cost": float(total),
       "projected_full_cost": float(total * scale_factor),
       "ci_low": float(ci_low),
       "ci_high": float(ci_high),
       "confidence_level": confidence_level,
       "n_bootstrap": n_bootstrap,
       "per_condition_breakdown": breakdown_list,
   }
   ```
9. Write to output_path as JSON
10. Log a human-readable summary table to logger.info showing per-condition costs
11. Return the dict

Add the following functions to `src/pilot.py`:

**check_preproc_fidelity(conn, pilot_prompt_ids, threshold=0.85) -> dict:**

1. Query completed runs where intervention is in ("pre_proc_sanitize", "pre_proc_sanitize_compress", "compress_only") and prompt_id is in pilot_prompt_ids
2. For each such run, the `prompt_text` column contains the PRE-PROCESSED text (after intervention). We need the original clean prompt text from prompts_by_id (accept prompts_by_id as parameter too).
3. Collect pairs: originals = [prompts_by_id[r["prompt_id"]]["prompt_text"] for r in preproc_runs], preprocessed = [r["prompt_text"] for r in preproc_runs]
4. If no pairs found, return early with empty results
5. Call `bert_score.score(preprocessed, originals, lang="en", verbose=False)` to get P, R, F1 tensors
6. Convert F1 to list, flag any below threshold
7. Return dict matching the pattern from 04-RESEARCH.md:
   ```python
   {
       "mean_f1": float, "min_f1": float, "threshold": threshold,
       "flagged_count": int, "total_pairs": int,
       "flagged_pairs": [{"index": i, "f1": score, "prompt_id": pid, "intervention": intv, "model": model}]
   }
   ```
8. NOTE: BERTScore downloads a ~400MB model on first run. This is expected. The function should handle ImportError gracefully and return a dict with `{"error": "bert-score not installed"}` if the package is missing.

**profile_latency(conn, pilot_prompt_ids) -> dict:**

1. Query completed pilot runs
2. Group by model and by (model, intervention)
3. For each group, compute statistics on ttft_ms and ttlt_ms: mean, median (p50), p95, max, min
4. Use numpy for percentile calculations: `np.percentile(values, [50, 95])`
5. Estimate wall-clock time for full run: `total_items * mean_ttlt_ms / 1000 / 3600` hours (rough, does not account for parallelism)
6. Flag any model+intervention combo where p95 ttlt > 30000ms (30 seconds)
7. Return:
   ```python
   {
       "by_model": {model: {"ttft": stats, "ttlt": stats}},
       "by_condition": {f"{model}|{intervention}": {"ttft": stats, "ttlt": stats}},
       "estimated_full_run_hours": float,
       "latency_flags": [...]
   }
   ```

**estimate_power(conn, pilot_prompt_ids) -> dict:**

1. Query completed pilot runs
2. Compute pass rate for clean+raw condition vs. each noisy+raw condition (e.g., type_a_20pct+raw)
3. Calculate observed effect size as difference in pass rates (clean_rate - noisy_rate)
4. Use a simplified binomial power calculation: for each observed effect, compute required N using `scipy.stats.norm` for z-test on two proportions
5. Formula: N = ((z_alpha + z_beta)^2 * (p1*(1-p1) + p2*(1-p2))) / (p1 - p2)^2, with alpha=0.05, power=0.80
6. Report whether N=200 is sufficient for the observed effect sizes
7. Return dict: `{"observed_effects": [...], "required_n": [...], "n_200_sufficient": bool, "note": "Rough estimate; full GLMM power analysis in Phase 5"}`

**run_pilot_verdict(conn, pilot_prompt_ids, prompts_by_id, budget=200.0, output_path="results/pilot_verdict.json") -> dict:**

1. Compute completion_rate: count completed / count total (completed + failed) for pilot prompt_ids
2. Check for systematic failures: group by model, intervention -- if any group has 0% completion, flag as systematic_failure
3. Check zero_variance: for each prompt_id + noise_type + intervention + model combo, check if all 5 repetitions have the same pass_fail value. Report % of conditions with zero variance. This is informational, not auto-fail.
4. Run all validation checks:
   - data_audit = audit_data_completeness(conn, pilot_prompt_ids)
   - noise_check = verify_noise_rates(prompts_by_id, pilot_prompt_ids)
   - cost_proj = compute_cost_projection(conn, pilot_prompt_ids)
   - budget_check = check_budget_gate(cost_proj["projected_full_cost"], budget)
   - spot_check = run_spot_check(conn, pilot_prompt_ids, prompts_by_id)
   - latency = profile_latency(conn, pilot_prompt_ids)
   - power = estimate_power(conn, pilot_prompt_ids)
   - Try BERTScore fidelity (catch ImportError)
5. Determine overall verdict:
   - FAIL if completion_rate < 0.95
   - FAIL if any systematic_failure detected
   - PASS otherwise (budget warning is informational, not auto-fail)
6. Build verdict dict:
   ```python
   {
       "generated_at": ...,
       "overall_verdict": "PASS" or "FAIL",
       "completion_rate": float,
       "systematic_failures": [...],
       "zero_variance_pct": float,
       "data_audit": data_audit,
       "noise_check": noise_check,
       "cost_projection": cost_proj,
       "budget_gate": budget_check,
       "spot_check_summary": {"total_sampled": N, "gsm8k_count": N, "code_count": N},
       "latency": latency,
       "power_analysis": power,
       "preproc_fidelity": fidelity_result or None,
       "flagged_issues": [...],
   }
   ```
7. Write to output_path as JSON with indent=2
8. Log the verdict summary (PASS/FAIL, completion rate, projected cost, key flags)
9. Return the verdict dict
