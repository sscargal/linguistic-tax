  <action>
Create src/analyze_results.py with these exact functions:

1. **load_experiment_data(db_path: str) -> pd.DataFrame:**
   - `pd.read_sql_query("SELECT * FROM experiment_runs WHERE status = 'completed' AND pass_fail IS NOT NULL", conn)`
   - Add derived columns: `noise_level_str` extracted from noise_type (e.g., "10" from "type_a_10pct", None for clean/type_b), `benchmark_source` derived from benchmark column

2. **fit_glmm(df: pd.DataFrame) -> dict:**
   - GLMM formula (2-way interactions only, NOT the * operator which creates higher-order):
     ```
     pass_fail ~ C(noise_type) + C(intervention) + C(model) + C(benchmark)
       + C(noise_type):C(intervention) + C(noise_type):C(model)
       + C(intervention):C(model)
     ```
   - Random effects: `vc_formulas = {"prompt_intercept": "0 + C(prompt_id)", "prompt_model": "0 + C(prompt_id):C(model)"}`
   - Three-level fallback chain:
     a. BinomialBayesMixedGLM.from_formula with full vc_formulas -> fit_vb()
     b. On Exception: drop "prompt_model" from vc_formulas, retry
     c. On Exception: GEE.from_formula with groups="prompt_id", family=sm.families.Binomial(), cov_struct=sm.cov_struct.Exchangeable()
   - Extract results: coefficient names, estimates (fe_mean for BayesMixedGLM, params for GEE), std errors, p-values
   - Compute odds_ratios: np.exp(estimates) with CIs via np.exp(estimate +/- 1.96*std_err)
   - Compute risk_differences: approximate via derivative of logistic CDF at mean
   - Return dict with: model_type ("bayesian_glmm", "reduced_glmm", "gee"), coefficients (list of dicts with name/estimate/std_err/p_value/odds_ratio/ci_lower/ci_upper), convergence_info, formula_used

3. **compute_bootstrap_cis(df: pd.DataFrame, n_iterations: int = 10000, seed: int = 42) -> dict:**
   - Group by (noise_type, intervention, model)
   - For each group, compute bootstrap CI for: accuracy (mean pass_fail), mean_cost (mean total_cost_usd)
   - Use scipy.stats.bootstrap with method='bca', n_resamples=n_iterations, random_state=seed
   - Fallback to method='percentile' on DegenerateDataWarning (same pattern as pilot.py)
   - Also compute bootstrap CIs for CR values from derived_metrics table
   - Return dict keyed by condition string, each with mean, ci_lower, ci_upper, method_used

4. **run_mcnemar_analysis(df: pd.DataFrame, baseline_condition: str = "clean", model_filter: str | None = None) -> dict:**
   - For each model (or model_filter):
     - For each noisy condition vs baseline:
       - For each prompt: build 2x2 table from paired repetitions (rep 1 vs rep 1, etc.)
       - Skip if b + c = 0 (concordant)
       - Run statsmodels.stats.contingency_tables.mcnemar(table, exact=True)
       - Classify: b > c = "fragile" (passes baseline, fails noisy), c > b = "recoverable"
   - Collect all p-values per family
   - Return dict with per-comparison results (prompt_id, statistic, p_value, classification, b_count, c_count), skipped_count

5. **compute_kendall_tau(df: pd.DataFrame) -> list[dict]:**
   - For each model:
     - Compute per-prompt pass rate under clean condition
     - For each noisy condition:
       - Compute per-prompt pass rate under noisy condition
       - Align on common prompts
       - scipy.stats.kendalltau(clean_rates, noisy_rates, variant='b')
       - Return dict: clean_condition, noisy_condition, model, tau, p_value, n_prompts

6. **apply_bh_correction(results_by_family: dict[str, list[float]], alpha: float = 0.05) -> dict:**
   - Input: {"mcnemar": [p1, p2, ...], "glmm": [p1, ...], "kendall": [p1, ...]}
   - For each family: multipletests(p_values, alpha=alpha, method='fdr_bh')
   - Return: {"mcnemar": {"raw_p_values": [...], "corrected_p_values": [...], "reject": [...]}, ...}

7. **run_sensitivity_analysis(df: pd.DataFrame, drop_pct: float = 0.10) -> dict:**
   - Compute per-prompt pass rate across all conditions
   - Sort prompts by overall pass rate
   - Drop bottom drop_pct (hardest) and top drop_pct (easiest)
   - Rerun fit_glmm and compute_bootstrap_cis on filtered data
   - Return dict with: n_prompts_original, n_prompts_filtered, n_dropped, glmm_results, bootstrap_results

8. **generate_effect_size_summary(glmm_results: dict, kendall_results: list[dict], bootstrap_results: dict) -> pd.DataFrame:**
   - Collect from glmm_results: OR, RD for each coefficient with CIs
   - Collect from kendall_results: tau values with p-values
   - Compute Cohen's d for token savings and latency differences between conditions (using Welch's correction: pooled_sd = sqrt((s1^2 + s2^2) / 2))
   - Return DataFrame with columns: metric, comparison, effect_size, ci_lower, ci_upper, effect_type (OR/RD/d/tau)

9. **main() -> None:**
   - argparse with --db (default "results/results.db"), --output-dir (default "results"), --seed (default 42), --bootstrap-iterations (default 10000), --sensitivity-drop-pct (default 0.10)
   - Subcommands: glmm, mcnemar, bootstrap, kendall, sensitivity, all
   - Each subcommand:
     a. Runs the corresponding analysis function
     b. Applies BH correction to the p-values produced
     c. Writes JSON to {output_dir}/{analysis}_results.json
     d. Writes CSV to {output_dir}/csv/{analysis}_*.csv
     e. Prints terminal summary via tabulate(format="grid")
   - "all" subcommand runs everything, produces analysis_summary.json and effect_size_summary.csv
   - if __name__ == "__main__": main()

All functions use logging module (no print). All functions have type hints and docstrings. Fixed random seeds for reproducibility.
