**Section 2: Literature Survey** -- Organize findings by these 6 format categories (per CONTEXT.md locked decision). For each category write 2-4 paragraphs covering: what the format is, key papers and findings, token efficiency data, accuracy impact data, and relevance to the Linguistic Tax paper.

Categories and key content for each:

1. **Token-Optimized Notations (TOON, CompactPrompt)**
   - TOON: 40% fewer tokens than JSON, 76.4% accuracy vs JSON 75.0% on data retrieval across 4 models (cite github.com/toon-format/toon)
   - CompactPrompt (ArXiv:2510.18043): 60% token reduction with less than 5% accuracy drop -- already cited in RDD
   - LLMLingua (ArXiv:2310.05736): up to 20x compression via perplexity-based token pruning, but aggressive compression degrades quality
   - Conservative 2-3x compression is the sweet spot per 2025 surveys
   - Note TOON benchmarks are data retrieval, NOT reasoning tasks -- flag as open question

2. **Structured Markup (XML, HTML)**
   - Anthropic official docs recommend XML tags for Claude prompts
   - He et al. (ArXiv:2411.10541): up to 40% performance variation from format choice; model-specific preferences with IoU below 0.2 between model families
   - XML adds tokens for tags but may improve parsing accuracy by providing explicit structure
   - Note: GPT-3.5 prefers JSON, GPT-4 prefers Markdown (He et al.) -- format preferences are NOT universal

3. **Minimal/Telegraphic (Bullet Points, Outline Format)**
   - Bullet formats generally outperform prose for option presentation
   - Training data bias likely explains this (StackOverflow, docs use bullets)
   - Effect size for coding instructions and math word problems is unknown -- flag as testable
   - Strips prose connectors ("therefore", "in order to") which are pure formatting overhead

4. **Punctuation Variations**
   - LLM-Microscope (ArXiv:2502.15007): punctuation tokens serve as attention sinks and carry context memory in transformers -- removal degrades MMLU and BABILong-4k
   - "When Punctuation Matters" (ArXiv:2508.11383): 8 models, 52 tasks confirm punctuation sensitivity is real
   - Punctuation and Predicates (ArXiv:2508.14067): model-specific punctuation necessity (GPT-2 vs DeepSeek vs Gemma)
   - KEY FINDING: Punctuation removal is NOT a simple token-saving win -- it may HURT accuracy. Frame as hypothesis to test, not assumed benefit.

5. **Hybrid/Novel Approaches**
   - CFPO (ArXiv:2502.04295): joint content AND format optimization outperforms content-only optimization -- format is a first-class optimization dimension
   - Lossless Meta-Tokens (ArXiv:2506.00307): 27% lossless compression via LZ77-like technique at token level
   - ProCut (ArXiv:2508.02053): Shapley-value-based compression identifies which tokens contribute to output quality
   - LLM-DCP (ArXiv:2504.11004): MDP-based dynamic compression, 17% improvement over Selective Context
   - These represent cutting-edge approaches that could inspire new intervention types

6. **Verbose Natural Language Baseline**
   - This is the current default in our experiment suite (the "raw" intervention)
   - Serves as the control condition for all format comparisons
   - The paper's thesis: verbose NL carries a "linguistic tax" of redundancy, filler words, and formatting overhead
   - Quantify: what fraction of typical prompt tokens are "structural" (punctuation, connectors, formatting) vs "semantic" (task-relevant content)?
