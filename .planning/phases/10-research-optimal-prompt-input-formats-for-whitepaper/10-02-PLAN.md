**Section 5: Testable Hypotheses** -- Write 6 hypotheses using the template from RESEARCH.md. The 6 hypotheses (IDs match H-FMT-NN pattern):

**H-FMT-01: TOON-like Compact Notation for Structured Code Prompts** (Priority: HIGH)
- Claim: Reformatting structured prompt data from JSON/verbose NL to TOON-like compact notation reduces input tokens by 30-40% while maintaining coding task accuracy on HumanEval/MBPP.
- Independent Variable: Prompt data format (verbose NL baseline vs TOON-like compact)
- Dependent Variables: Pass rate, input token count, cost
- Benchmarks: HumanEval + MBPP subset (prompts with structured context/constraints)
- Models: Claude Sonnet, Gemini 1.5 Pro (same as main experiment)
- Sample Size: 20 prompts x 5 repetitions x 2 models x 2 formats = 400 API calls
- Estimated Cost: $3-8 (based on current PRICE_TABLE rates)
- Expected Effect: 30-40% token reduction, less than 5% accuracy change
- Measurement: Paired comparison -- same prompt in both formats, McNemar's test for accuracy difference, token count ratio
- Supporting Evidence: TOON benchmarks show 76.4% vs JSON 75.0% on data retrieval; CompactPrompt achieves 60% reduction with less than 5% accuracy drop
- Risks: TOON benchmarks are on data retrieval, not reasoning. Effect may not transfer to code generation.

**H-FMT-02: XML Structured Markup for Claude vs Gemini** (Priority: HIGH)
- Claim: XML-tagged prompt structure improves Claude accuracy by 5-10% (Anthropic recommends XML) but has no effect or negative effect on Gemini, demonstrating model-specific format preferences.
- Independent Variable: Prompt structure (plain text vs XML-tagged sections)
- Dependent Variables: Pass rate, input token count, TTFT
- Benchmarks: Full benchmark set (HumanEval, MBPP, GSM8K) -- 20-prompt pilot subset
- Models: Claude Sonnet, Gemini 1.5 Pro
- Sample Size: 20 prompts x 5 repetitions x 2 models x 2 formats = 400 API calls
- Estimated Cost: $3-8
- Expected Effect: Claude +5-10% accuracy, Gemini +/-2% (neutral), 5-15% token increase from XML tags
- Measurement: Per-model accuracy comparison, interaction effect in GLMM (format x model)
- Supporting Evidence: He et al. show IoU below 0.2 between model format preferences; Anthropic docs recommend XML for Claude
- Risks: XML overhead may negate accuracy gains on cost basis.

**H-FMT-03: Bullet/Outline Format for Math Word Problems** (Priority: MEDIUM)
- Claim: Converting GSM8K prose word problems to bullet-point format (key values + question) reduces tokens by 15-25% and maintains or improves accuracy.
- Independent Variable: Instruction format (prose paragraph vs bulleted key information)
- Dependent Variables: Pass rate, input token count
- Benchmarks: GSM8K subset (20 prompts with verbose word problems)
- Models: Claude Sonnet, Gemini 1.5 Pro
- Sample Size: 20 prompts x 5 repetitions x 2 models x 2 formats = 400 API calls
- Estimated Cost: $2-5
- Expected Effect: 15-25% token reduction, less than 3% accuracy change
- Measurement: Paired comparison, exact-match grading (same as current GSM8K grading)
- Supporting Evidence: Bullet formats outperform prose for option presentation in training data; telegraphic style removes filler words
- Risks: Math word problems may encode reasoning cues in prose connectors ("therefore", "so") that bullets strip.

**H-FMT-04: Punctuation Removal Effects on Coding Tasks** (Priority: HIGH)
- Claim: Removing "optional" punctuation (trailing periods in instructions, commas in lists, semicolons) from coding prompts degrades accuracy by 2-8%, contrary to the naive token-saving hypothesis.
- Independent Variable: Punctuation level (full punctuation vs stripped punctuation)
- Dependent Variables: Pass rate, input token count, consistency rate (CR)
- Benchmarks: HumanEval + MBPP subset (20 prompts)
- Models: Claude Sonnet, Gemini 1.5 Pro
- Sample Size: 20 prompts x 5 repetitions x 2 models x 2 formats = 400 API calls
- Estimated Cost: $3-8
- Expected Effect: 5-10% token reduction BUT 2-8% accuracy DECREASE
- Measurement: Paired comparison, CR comparison (does punctuation removal also hurt stability?)
- Supporting Evidence: LLM-Microscope shows punctuation as attention sinks; "When Punctuation Matters" confirms across 8 models and 52 tasks
- Risks: Effect may be smaller on coding tasks than on MMLU-style tasks where punctuation was studied. This is a genuinely uncertain hypothesis.

**H-FMT-05: Format x Noise Interaction (Novel Contribution)** (Priority: MEDIUM)
- Claim: Structured formats (XML, bullets) make prompts MORE robust to Type A character-level noise because structure tokens survive noise injection better than prose connectors.
- Independent Variable: Format (prose vs XML vs bullet) crossed with noise level (0%, 5%, 10%, 20%)
- Dependent Variables: Pass rate, accuracy degradation slope
- Benchmarks: HumanEval subset (20 prompts)
- Models: Claude Sonnet, Gemini 1.5 Pro
- Sample Size: 20 prompts x 5 reps x 2 models x 3 formats x 4 noise levels = 2,400 API calls
- Estimated Cost: $15-30
- Expected Effect: Structured formats show shallower accuracy degradation curve with increasing noise
- Measurement: GLMM with format x noise_level interaction term; compare degradation slopes
- Supporting Evidence: No direct evidence -- this is a NOVEL hypothesis. Motivated by: (a) structural tokens like XML tags are longer and may survive character mutations better, (b) prose connectors are high-entropy and vulnerable to noise
- Risks: Most expensive hypothesis. Consider as Phase 11 stretch goal. Noise injection on XML tags could break structure entirely (e.g., `<task>` becomes `<tsak>`), which would be worse than prose degradation.

**H-FMT-06: Question Mark Presence for Query Prompts** (Priority: LOW)
- Claim: Including vs omitting a question mark at the end of a query prompt has a measurable (but small) effect on response quality, particularly for GSM8K math questions.
- Independent Variable: Terminal question mark (present vs absent)
- Dependent Variables: Pass rate
- Benchmarks: GSM8K subset (20 prompts that are questions)
- Models: Claude Sonnet, Gemini 1.5 Pro
- Sample Size: 20 prompts x 5 repetitions x 2 models x 2 formats = 400 API calls
- Estimated Cost: $2-5
- Expected Effect: Less than 2% accuracy difference (may be noise)
- Measurement: McNemar's test for significance
- Supporting Evidence: "When Punctuation Matters" suggests punctuation carries signal, but question marks specifically are understudied. This is a micro-formatting probe.
- Risks: Effect size may be too small to detect with 20 prompts. Consider as a quick-add to another experiment rather than standalone.
