**H-FMT-04 Punctuation decomposition — break into per-punctuation-type atomic questions:**
- AQ-PM-01: Remove trailing periods from instruction sentences in HumanEval prompts (claim: period removal degrades accuracy 2-5% because periods serve as sentence boundary markers for attention)
- AQ-PM-02: Remove commas from lists and clauses in coding prompts (claim: comma removal hurts parameter list parsing, 3-8% accuracy decrease)
- AQ-PM-03: Remove semicolons from multi-clause instructions (claim: semicolons serve as clause boundaries; removal has smaller effect than periods, 1-3%)
- AQ-PM-04: Remove ALL "optional" punctuation simultaneously from HumanEval prompts (claim: combined removal has larger effect than individual, 5-10% accuracy decrease)
- AQ-PM-05: Remove ALL "optional" punctuation from GSM8K math problems (claim: math problems may be less sensitive than coding since reasoning structure is in numbers, not punctuation)
- AQ-PM-06: Partial punctuation removal — remove only end-of-sentence periods but keep commas/semicolons (claim: isolates the sentence boundary effect from clause-level effects)

**H-FMT-06 Question mark decomposition:**
- AQ-PM-07: Remove terminal question mark from GSM8K questions (claim: less than 2% accuracy difference; question mark is largely redundant when the prompt is clearly a question)
- AQ-PM-08: Remove question mark AND rephrase as imperative ("What is X?" -> "Calculate X.") (claim: rephrasing is a confound — isolate the question mark effect by comparing "What is X?" vs "What is X")

**Note on bundling:** Per CONTEXT.md specifics, H-FMT-06 was recommended to bundle with H-FMT-04 rather than standalone.   - Semicolons: `re.sub(r';\s', ' ', text)` — removes semicolons
  - Question marks: `re.sub(r'\?\s*$', '', text)` — removes terminal question mark
  - ALL specs must note: preserve code blocks using the code-block placeholder pattern from the research

**File header:**
- Title: "Format x Noise Interaction Experiments: Does Structure Protect Against Noise?"
- Parent hypothesis: H-FMT-05
- Date, status (Draft)
- Novel contribution note: "No existing paper examines whether structured formatting makes prompts more or less robust to character-level noise. This is a potential whitepaper-differentiating finding."

**H-FMT-05 decomposition — create atomic questions for each format x noise combination:**
- AQ-FN-01: XML-structured prompts x Type A noise at 5/10/20% on HumanEval (claim: XML tags survive character mutations better than prose because tags are multi-character sequences; structured prompts show shallower accuracy degradation slope)
- AQ-FN-02: Bullet-formatted prompts x Type A noise at 5/10/20% on HumanEval (claim: bullet markers (-, *, 1.) are short single-token markers that may be MORE vulnerable to noise than prose connectors)
- AQ-FN-03: TOON-formatted prompts x Type A noise at 5/10/20% on HumanEval (claim: compact notation has fewer redundant tokens, so each corrupted token has higher information density — noise may hurt MORE in compact formats)
- AQ-FN-04: XML-structured prompts x Type B ESL noise on HumanEval (claim: syntactic noise targets prose not markup, so XML structure should be unaffected by ESL patterns)
- AQ-FN-05: Micro-pilot gate experiment — 5 prompts x 3 formats (prose, XML, bullet) x 4 noise levels (0, 5, 10, 20%) x 5 reps = 300 calls (claim: determines whether the full 2,400-call experiment is worth running)

**CRITICAL: Micro-pilot gate per CONTEXT.md decision:**
The CONTEXT.md locked decision states: "H-FMT-05 (format x noise, 2,400 calls): include with micro-pilot gate -- 60-call micro-pilot first, go/no-go decision before full experiment."

- The other AQ-FN-01 through AQ-FN-04 are the FULL experiments that run only if the micro-pilot shows signal

- Statistical analysis: GLMM with format x noise_level interaction term (already implemented in analyze_results.py). Compare degradation SLOPES across formats, not just point estimates.
- Cost: AQ-FN-05 micro-pilot = $0 on free models (300 calls x $0). Full experiments (AQ-FN-01 through AQ-FN-04) = $0 on free models, $15-30 on paid models.
