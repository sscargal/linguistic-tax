**SECTION 1: Full Hypothesis Specs (top 3-5 ideas)**

Select the 3-5 most promising ideas from the CONTEXT.md brainstorming lists and give each a full atomic experiment spec (AQ-NH-{NN} format, same template as Plans 01/02). Recommended top picks based on literature evidence:

1. **AQ-NH-01: Imperative vs Interrogative vs Declarative instruction phrasing** (from "Instruction phrasing" category)
   - "Write a function..." vs "Can you write a function...?" vs "The function should..."
   - Strong candidate: clear variable isolation, easy regex conversion, applies to all benchmarks
   - Benchmarks: HumanEval + MBPP (coding instructions have clear phrasing patterns)
   - Expected: Imperative performs best (training data bias toward imperative instructions in StackOverflow/docs)

2. **AQ-NH-02: Politeness markers ("Please"/"Thank you") effect on coding accuracy** (from "Instruction phrasing" category)
   - "Write a function..." vs "Please write a function..." vs "Please write a function... Thank you."
   - Supported by 3 papers: ArXiv:2510.04950 (impolite > polite on GPT-4o), ArXiv:2512.12812 (neutral/friendly > rude), ArXiv:2402.14531 (varies by language)
   - Benchmarks: HumanEval + MBPP
   - Expected: Mixed — literature disagrees, making this genuinely interesting

3. **AQ-NH-03: Code comment presence in example code** (from "Code-specific formatting" category)
   - Prompts with commented code examples vs stripped-comment code examples
   - Supported by Pan et al. (ArXiv:2508.13666): removing code formatting saves 24.5% tokens with <1% accuracy change
   - Benchmarks: HumanEval + MBPP (code-only)
   - Expected: Comment removal saves tokens with negligible accuracy impact

4. **AQ-NH-04: Newline density between prompt sections** (from "Whitespace/layout" category)
   - Single newline vs double newline vs no newline between instruction sections
   - Supported by Pan et al.: newlines are the biggest token contributor for Claude and Gemini
   - Benchmarks: All three (HumanEval, MBPP, GSM8K)
   - Expected: Reducing newlines saves 5-10% tokens with no accuracy change

5. **AQ-NH-05: Emphasis markers on key terms** (from "Structural markers" category)
   - Key terms unmarked vs **bold** vs CAPS vs 'quoted'
   - Supported by markdown structure literature: formatting helps LLMs parse content
   - Benchmarks: HumanEval + MBPP (emphasize function name, return type, key constraints)
   - Expected: Some emphasis may help, but CAPS could confuse (associated with shouting/urgency in training data)
