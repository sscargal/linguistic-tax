    - `PROVIDERS` dict mapping provider key to {name, models, env_var}:
      ```python
      PROVIDERS = {
          "anthropic": {
              "name": "Anthropic (Claude)",

              "models": [m for m in MODELS if m.startswith("gpt")],
              "env_var": "OPENAI_API_KEY",
          },
          "openrouter": {
              "name": "OpenRouter (free models)",
              "models": [m for m in MODELS if m.startswith("openrouter/")],
              "env_var": "OPENROUTER_API_KEY",
          },
      }
      ```

    - `REQUIRED_PACKAGES`: list of strings: `["anthropic", "google-genai", "openai", "scipy", "statsmodels", "pandas", "matplotlib", "pytest"]`

    - `check_environment() -> list[tuple[str, bool, str]]`: Checks Python >= 3.11 using sys.version_info. Checks each REQUIRED_PACKAGES using importlib.metadata.version(). Returns list of (check_name, passed, detail_string).

    - `validate_api_key(provider: str, env_var: str) -> tuple[bool, str]`: Gets key from os.environ.get(env_var). If not set, returns (False, f"Environment variable {env_var} is not set"). Then makes minimal API call based on provider:
      - "anthropic": `anthropic.Anthropic(api_key=key).messages.create(model="claude-haiku-4-5-20250514", max_tokens=1, messages=[{"role": "user", "content": "Hi"}])`
      - "google": `genai.Client(api_key=key).models.generate_content(model="gemini-2.0-flash", contents="Hi", config={"max_output_tokens": 1})`
      - "openai": `openai.OpenAI(api_key=key).chat.completions.create(model="gpt-4o-mini-2024-07-18", max_tokens=1, messages=[{"role": "user", "content": "Hi"}])`
      - "openrouter": `openai.OpenAI(api_key=key, base_url=OPENROUTER_BASE_URL).chat.completions.create(model="nvidia/nemotron-3-nano-30b-a3b:free", max_tokens=1, messages=[{"role": "user", "content": "Hi"}])`
      On success: returns (True, "API key validated successfully").
      On exception: checks str(e).lower() for "401", "403", "invalid", "auth" -- if found returns (False, f"Authentication failed: {e}"). Otherwise returns (False, f"API call failed (key may be valid, but got error): {e}").

    - `run_setup_wizard(args) -> None`: The main wizard function. Uses `print()` for interactive prompts (not logging -- see CONTEXT.md/RESEARCH.md guidance).

      If `args.non_interactive`: calls `get_full_config_dict()`, writes with `save_config()`, prints path, returns.

      Otherwise, interactive flow:
      1. Print welcome: "Welcome to the Linguistic Tax slicer setup wizard!"
      2. **Environment check**: call check_environment(), print results as pass/fail table. If any critical failures (Python version), warn but continue.
      3. **Provider selection**: Print numbered list of providers from PROVIDERS dict. Read input for selection (1-4). Map to provider key.
      4. **Target model**: Show models for selected provider. Default to first. Let user accept (Enter) or type number to override.
      5. **Preproc model**: Auto-fill from PREPROC_MODEL_MAP. Show and let user accept or override.
      6. **API key check**: Check if env var is set (os.environ.get). If set, offer to validate with test call. If not set, print instruction: f"Set {env_var} in your environment before running experiments."
      7. **Paths**: Show default paths (prompts_path, matrix_path, results_db_path). Let user accept defaults (Enter) or type new path for each.
      8. **Build config**: Start with `get_full_config_dict()`. Override with user selections (only the model fields for the chosen provider -- e.g., if anthropic, set claude_model).
      9. **Validate**: Call `validate_config()`. If errors, print them and ask user to fix or continue anyway.
      10. **Save**: Call `save_config()`. Print: f"Config saved to {path}"
      11. Print: "Setup complete! Run experiments with: python src/run_experiment.py"

      All input() calls should accept an `input_fn` parameter (defaulting to `builtins.input`) for testability. Implement this by having `run_setup_wizard` accept an optional `input_fn` keyword argument.
