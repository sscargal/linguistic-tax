**Functions:**

1. `_get_benchmark(prompt_id: str) -> str` -- Copy the same logic from run_experiment.py (HumanEval/ -> humaneval, Mbpp/ -> mbpp, gsm8k -> gsm8k, else -> unknown).

2. `estimate_cost(items: list[dict[str, Any]]) -> dict[str, float]` -- For each item, look up benchmark from `_get_benchmark(item["prompt_id"])`, get average tokens from AVG_TOKENS (default to humaneval if unknown), compute target model cost via `compute_cost(item["model"], tokens["input"], tokens["output"])`. For items with intervention in PREPROC_INTERVENTIONS, also compute pre-processor cost using `PREPROC_MODEL_MAP.get(item["model"], item["model"])` with pre-proc output tokens = int(tokens["input"] * 0.8). Return dict with keys "target_cost", "preproc_cost", "total_cost".

3. `estimate_runtime(items: list[dict[str, Any]]) -> float` -- Count items per model using Counter. For each model, multiply count by RATE_LIMIT_DELAYS.get(model, 0.5). Sum all to get total wall-clock seconds.

4. `count_completed(items: list[dict[str, Any]], conn: Any) -> tuple[int, int, list[dict[str, Any]]]` -- Import and call `query_runs(conn, status="completed")` to get completed run IDs. For each item, check if its run_id (built via same pipe-separated format as run_experiment.make_run_id) is in completed set. Return (completed_count, total_count, pending_items). Import make_run_id from src.run_experiment.

5. `format_summary(items: list[dict[str, Any]], completed_count: int, total_count: int, cost_estimate: dict[str, float], runtime_seconds: float) -> str` -- Build multi-section summary string using tabulate for alignment. Sections:
   - Header: "=== Pre-Execution Summary ==="
   - If resuming (completed_count > 0): "Resuming: {completed_count} of {total_count} done, {len(items)} remaining"
   - Models section: table with columns [Model, Items, Est. Cost] using Counter on item["model"]
   - Interventions section: table with columns [Intervention, Items] using Counter on item["intervention"]
   - Noise Conditions section: table with columns [Noise Type, Items] using Counter on item["noise_type"]
   - Cost section: "Target model cost: ${target_cost:.2f}", "Pre-processor cost: ${preproc_cost:.2f}", "Total estimated cost: ${total_cost:.2f}"
   - Runtime section: "Estimated runtime: {formatted_time}" (convert seconds to Xh Ym Zs format)
   - Use tabulate with "simple" format for tables. No ASCII bar charts per user decision.

6. `confirm_execution(summary: str, yes: bool = False, budget: float | None = None, estimated_cost: float = 0.0, input_fn: Callable[[str], str] = input) -> str` -- Print the summary string. Check budget FIRST: if budget is not None and estimated_cost > budget, print "WARNING: Estimated cost ${estimated_cost:.2f} exceeds budget ${budget:.2f}" and call `sys.exit(1)`. If yes=True, return "yes". Otherwise loop prompting with input_fn("[Y]es to run, [N]o to abort, [M]odify filters: "), parse y/yes -> "yes", n/no -> "no", m/modify -> "modify". Print "Invalid choice. Enter Y, N, or M." on bad input.

7. `save_execution_plan(items: list[dict[str, Any]], cost_estimate: dict[str, float], runtime_estimate: float, filters: dict[str, Any] | None = None, output_path: str = "results/execution_plan.json") -> None` -- Build a dict with keys: "timestamp" (ISO format UTC), "total_items" (len(items)), "models" (sorted unique models), "interventions" (sorted unique interventions), "noise_types" (sorted unique noise types), "cost_estimate" (the dict), "runtime_estimate_seconds" (float), "filters" (the filters dict or {}). Create parent directory if needed. Write as JSON with indent=2.

All functions must have type hints and docstrings. Use `from __future__ import annotations` for modern type syntax. Use logging module for debug messages only.
