**Default model lookup:** Build defaults from registry -- for each provider in PROVIDER_ORDER, find the first target model from `registry.target_models()` whose model_id starts with the provider prefix (claude->anthropic, gemini->google, gpt->openai, openrouter/->openrouter). Store in `DEFAULT_TARGET_MODELS: dict[str, str]`.

**Helper functions (all with type hints and docstrings):**

1. `_mask_key(key: str) -> str` -- If len(key) <= 10 return "****", else return `f"{key[:6]}...{key[-4:]}"`.

2. `_parse_provider_selection(raw: str, available: list[str]) -> list[str]` -- Parse comma-separated numbers (1-indexed) into provider keys from `available` list. Invalid entries skipped. If result is empty, default to `[available[0]]`.

3. `_detect_existing_config() -> dict | None` -- Call `find_config_path()`. If found, load the JSON, extract the `models` list, group by provider, return `{"providers": list[str], "models": dict[str, dict]}` where models maps provider to `{"target": model_id, "preproc": model_id}`. Return None if no config file.

4. `_handle_existing_config(existing: dict, input_fn: Callable) -> str` -- Print current providers/models, prompt: "What would you like to do?\n  1. Add a provider\n  2. Reconfigure everything\n  3. Start fresh\nChoice [2]: ". Return "add", "reconfigure", or "fresh". Default to "reconfigure" on empty input.

5. `_select_providers(input_fn: Callable, existing_providers: list[str] | None = None) -> list[str]` -- Show numbered list of 4 providers with "(configured)" marker for existing. Prompt: "Enter numbers separated by commas (e.g., 1,3): ". Parse with `_parse_provider_selection`. Return list of provider keys.

6. `_collect_api_keys(providers: list[str], input_fn: Callable, env_path: Path | None = None) -> dict[str, str]` -- For each provider: check `os.environ` for existing key. If exists, show masked key with "Keep this? (Y/n): ". If user says no or key missing, prompt for new key. Call `write_env(env_var, new_key, env_path=env_path)` then set `os.environ[env_var] = new_key` immediately. Return dict mapping provider to key value.

7. `_explain_model_roles(input_fn: Callable) -> str` -- Print explanation:
   ```
   Target models are the LLMs being tested in the experiment. These are the
   models whose accuracy under noisy prompts you want to measure.

   Pre-processor models are cheap, fast models (like Haiku or Flash) that
   clean up prompts before they reach the target model. They run on every
   prompt, so cost matters.
   ```
   Then ask: "Use a separate pre-processor per provider, or one global pre-processor?\n  1. Per-provider (default)\n  2. Global\nChoice [1]: ". Return "per-provider" or "global".

8. `_select_models(providers: list[str], preproc_scope: str, input_fn: Callable, existing_models: dict | None = None) -> list[dict]` -- For each provider, prompt for target model with default shown: `"Target model for {PROVIDER_NAMES[provider]} [{default}]: "`. If user types "list", call `_browse_models(provider, input_fn)` and use returned model_id. Then handle preproc: if preproc_scope=="per-provider", auto-assign preproc from `registry.get_preproc(target)` or use same model if no mapping. Show: "Pre-processor: {preproc} (auto-assigned). Enter to accept or type model ID: ". Collect free-text override. For preproc_scope=="global", only ask for preproc on the first provider, reuse for all. Return list of `{"provider": str, "target_model": str, "preproc_model": str}`.

9. `_browse_models(provider: str, input_fn: Callable, timeout: float = 5.0) -> str | None` -- Import and call the per-provider query function from model_discovery (`_query_anthropic`, `_query_google`, `_query_openai`, `_query_openrouter`). Map provider to the correct function. If query fails or returns empty, call `_get_fallback_models(provider)` and print "(Using cached model list -- live API unreachable)". Sort models by model_id descending. Paginate 20 per page. Show: `"  {i}. {model_id} ({context_window:,} tokens)"` for each (omit context window if None). Footer: `"Page {page}/{total} | n=next p=prev #=select /text=search q=quit"`. Handle: 'n'=next page, 'p'=prev page, number=select that model's model_id, '/text'=set substring filter (case-insensitive), 'q'=return None.

10. `validate_api_key(provider: str, env_var: str, model_id: str | None = None) -> tuple[bool, str]` -- Same as current but add `model_id` parameter. Use `model_id` in the API call if provided, otherwise fall back to current hardcoded defaults. Keep all 4 provider branches (anthropic, google, openai, openrouter). Keep the auth-error vs other-error distinction.

11. `_validate_models(models: list[dict], input_fn: Callable) -> list[dict]` -- For each model entry, call `validate_api_key(provider, PROVIDER_ENV_VARS[provider], model_id=target_model)`. Print "Validating {target_model}... " then "OK" or "FAILED: {msg}". On failure, ask "Keep this model? (Y/n): ". If user says no, skip this model from results. Return filtered list.

12. `_build_budget_preview(models: list[dict]) -> str` -- Build synthetic experiment items for both pilot (20 prompts) and full (200 prompts) runs. For each model config, create items with all 5 interventions x 5 repetitions x prompt_count prompts. Distribute prompt_ids as: first 40% "HumanEval/{i}", next 40% "Mbpp/{i}", last 20% "gsm8k_{i}". Call `estimate_cost(items)` from execution_summary for both pilot and full. Format output:
   ```
   Budget Estimate
   ---------------
   Pilot run (20 prompts):
     {model}: ~${cost:.2f}
     ...
     Pre-processing: ~${preproc:.2f}
     Total: ~${total:.2f}

   Full run (200 prompts):
     {model}: ~${cost:.2f}
     ...
     Pre-processing: ~${preproc:.2f}
     Total: ~${total:.2f}
   ```
   For models where `registry.get_price(model_id) == (0.0, 0.0)` AND model_id is not in `registry._models`, show as "pricing unknown" instead of $0.00. If full run total exceeds $50, append "WARNING: Full run estimate exceeds $50.00". Include note "Total (excluding unknown): ~${known_total:.2f}" if any models have unknown pricing.

13. `_show_confirmation(models: list[dict], keys: dict[str, str], input_fn: Callable) -> bool` -- Print full summary table: provider, target model, preproc model, key status (set/not set). Run `validate_config()` on the config dict and print any warnings. Show budget preview (call `_build_budget_preview`). Ask "Save this configuration? (Y/n): ". Return True if user confirms.

14. `_build_config_dict(models: list[dict]) -> dict` -- Build config dict suitable for `save_config()`. Start with `get_full_config_dict()`. Set `config_version: 2`. Build `models` list of dicts matching the format in default_models.json: for each model entry, create target entry `{"model_id": target_model, "provider": provider, "role": "target", "preproc_model_id": preproc_model}` plus preproc entry `{"model_id": preproc_model, "provider": provider, "role": "preproc"}`. Look up pricing from registry for known models, set to None for unknown. Deduplicate preproc entries (same preproc used by multiple targets).

15. `run_setup_wizard(args: Any, input_fn: Callable[..., str] | None = None) -> None` -- Main entry point. Preserve signature exactly. Flow:
    - If `args.non_interactive`: write defaults via `get_full_config_dict()` + `save_config()`, return.
    - Wrap entire interactive flow in `try: ... except KeyboardInterrupt: print("\nSetup cancelled."); return`.
    - Print welcome banner.
    - Call `check_environment()`, display results.
    - Call `_detect_existing_config()`. If existing, call `_handle_existing_config()`. If "add", merge existing providers into flow. If "fresh", ignore existing.
    - Call `_select_providers()`.
    - Print "\nAPI Key Configuration\n" header.
    - Call `_collect_api_keys()`.
    - Call `_explain_model_roles()` to get preproc_scope.
    - Call `_select_models()`.
    - Call `_validate_models()`.
    - If no models remain after validation, print "No valid models configured. Exiting." and return.
    - Call `_show_confirmation()`. If declined, print "Configuration not saved." and return.
    - Build config via `_build_config_dict()`, call `save_config()`, call `registry.reload()` with new models.
    - Print post-setup next steps: "Next steps:\n  Run 'propt pilot' to test with 20 prompts\n  Run 'propt list-models' to see available models\n  Run 'propt config show' to review your configuration".

16. `check_environment() -> list[tuple[str, bool, str]]` -- Preserve exactly as-is from current implementation.
